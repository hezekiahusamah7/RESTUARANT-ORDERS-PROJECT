# 🍽️ Restaurant Orders Project

This project explores restaurant order data using SQL to uncover insights about menu performance, pricing, and customer behavior.

## 🔍 Key Findings and Analyses

### Phase 1: Menu Insights
- Identified the **most expensive Italian dishes**.
- Calculated the **average dish price** within each category.

### Phase 2: Order Exploration
- Viewed all tables using **JOIN queries**.
- Determined the **date range** of the dataset.
- Counted **total orders** and **items ordered** within that range.
- Found which **orders contained the most items**.
- Discovered how many **orders had more than 12 items**.

### Phase 3: Customer Behavior Analysis
- Combined `menu_items` and `order_details` tables using **JOIN** to analyze customer patterns.
- Determined the **least and most ordered items**, along with their categories.
- Identified the **top 5 orders** that spent the most money.
- Extracted additional valuable insights from the dataset.

---

This project demonstrates SQL techniques such as aggregation, filtering, and table joins to analyze real‑world restaurant data and derive actionable insights.
